## 1.如何KYC认证和投资人资格验证
```solidity
  // ===== KYC =====
  function setWhitelist(address user, bool ok) external onlyOwner {
      whitelisted[user] = ok;
  }
```
在智能合约中，我们定义了一个 `setWhitelist` 函数，这个函数允许合约的所有者（Owner）将一个地址添加到白名单中，或者从白名单中移除。这个函数接受两个参数：`user`（需要设置的地址）和 `ok`（一个布尔值，表示是否将该地址添加到白名单中）。
当投资人提交认购请求时，智能合约会检查他们的地址是否在白名单中，以确定他们是否已经完成了KYC认证并且符合投资资格。如果地址不在白名单中，智能合约会拒绝他们的认购请求，并且可能会返回一个错误信息，提示他们需要先完成KYC认证。
这个KYC认证的过程通常是在前端页面上完成的，投资人需要提供相关的身份信息和文件，经过审核后，管理员会将他们的地址添加到智能合约的白名单中。这样，在投资人提交认购请求时，智能合约就可以通过检查白名单来验证他们的资格。 
在前端页面上，我们可以设计一个KYC认证的流程，投资人需要上传相关的身份信息和文件，经过审核后，管理员会将他们的地址添加到智能合约的白名单中。这样，在投资人提交认购请求时，智能合约就可以通过检查白名单来验证他们的资格。
在后端数据库中，我们也可以维护一个投资人信息表，记录他们的KYC认证状态和相关信息。当投资人提交认购请求时，我们可以先查询数据库，确认他们的KYC认证状态是否有效，然后再调用智能合约的 `subscribe` 函数进行认购操作。这样可以确保只有符合资格的投资人才能参与认购，并且我们也可以在数据库中记录他们的投资行为和相关信息。

## 2. 典型设计
链上白名单 + 数据库同步 的典型设计
核心目标：

合约设置白名单

数据库保存白名单状态

API 调用合约

同步数据库

一个 完整架构方案（合约 + DB + API）
### 一、Solidity 合约函数

```solidity
function setWhitelist(address user, bool ok) external onlyOwner {
    whitelisted[user] = ok;
}
```
建议 加一个 Event，方便链上同步。
```solidity
event WhitelistUpdated(address indexed user, bool ok);

function setWhitelist(address user, bool ok) external onlyOwner {
    whitelisted[user] = ok;
    emit WhitelistUpdated(user, ok);
}
```
可以用 cron 扫事件 或 监听事件同步数据库

### 二、数据库设计
数据库中可以设计一个表来保存投资人的 KYC 认证状态，例如：

```sql
CREATE TABLE whitelist (
  id SERIAL PRIMARY KEY,
  wallet_address TEXT UNIQUE NOT NULL,
  is_whitelisted BOOLEAN DEFAULT false,
  updated_at TIMESTAMP DEFAULT now()
);
| 字段             | 说明    |
| -------------- | ----- |
| wallet_address | 钱包地址  |
| is_whitelisted | 是否白名单 |
| updated_at     | 更新时间  |

```

### 三、管理员 API 调用合约

Next.js API 路由示例：

```javascript
/api/rwa/whitelist
```

示例：
```ts
/* eslint-disable @typescript-eslint/no-explicit-any */
import { NextRequest, NextResponse } from "next/server";
import pool from "@/lib/db";
import { getRWAPlatformContract } from "@/lib/rwaPlatformContract";

export async function POST(req: NextRequest) {
  try {
    const { address, ok } = await req.json();

    if (!address) {
      return NextResponse.json({ error: "Missing address" }, { status: 400 });
    }

    const contract = getRWAPlatformContract();

    // 调用链上
    const tx = await contract.setWhitelist(address, ok);
    const receipt = await tx.wait();

    // 同步数据库
    await pool.query(
      `
      INSERT INTO whitelist (wallet_address, is_whitelisted)
      VALUES ($1,$2)
      ON CONFLICT (wallet_address)
      DO UPDATE SET
        is_whitelisted = EXCLUDED.is_whitelisted,
        updated_at = now()
      `,
      [address.toLowerCase(), ok],
    );

    return NextResponse.json({
      success: true,
      txHash: tx.hash,
      blockNumber: receipt.blockNumber,
    });
  } catch (err: any) {
    return NextResponse.json({ error: err.message }, { status: 500 });
  }
}
```

### 四、同步链上白名单（可选）
如果你担心：

数据库丢失

管理员直接链上操作

可以做 cron 同步事件

监听：
```
WhitelistUpdated(address user, bool ok)
```
示例：
```ts
const filter = contract.filters.WhitelistUpdated();

const logs = await contract.queryFilter(filter, fromBlock, toBlock);
```

解析：
```ts
for (const log of logs) {
  const parsed = iface.parseLog(log);

  const user = parsed.args.user;
  const ok = parsed.args.ok;

      updated_at = now()
    `,

  );
}
```
